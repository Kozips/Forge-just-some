## 第一步：加载写作引擎

**`写作引擎.md`** 是独立通用模块，包含：

- 通用原则（身体写作、角色声音、信息渗透、张力、节奏、结尾）
- 题材适配（恐怖/情感/搞笑/热血）
- 禁用速查（禁用词、AI 腔禁令、句式规则、情感替换、密度底线、对话规则）
- 写后自检

**`写作/写作流程/质检/05_写作风格自用.md`** 默认笔风文件，海明威口味——干净利落+神秘克制。**写任何正文前必须加载。**

**先读引擎和笔风，再回来查下面的文件导航。**

### 加载规则

- **先读 `写作引擎.md` → 再读题材 SKILL.md（铁律）→ 然后写。遇到具体需求时按需加载 1 个文件，不要预加载。**
- 题材文件命名规则固定，根据用户说的题材直接定位。
- 对话习惯文件单个可达 50k+ token，**禁止批量加载**，只读命中关键词的单个文件。
- 角色完整档案（3个MD合计 ~43k）只在长篇连载建角色时加载。
- **中文思考常开**：`规则/中文思考.md`（无论何时都按中文组织思考/分析）。
- **创作边界与声明**：`规则/创作声明.md`（按需查阅；本 skill 仅服务虚构创作）。

## 硬规则

### 冲突处理

| 冲突        | 处理                       |
| ----------- | -------------------------- |
| 恐怖+搞笑   | 允许，恐怖为主，搞笑为调剂 |
| 悬疑+恐怖   | 允许，组合加载             |
| 社恐+话痨   | 拒绝，选一个               |
| 腹黑+傻白甜 | 可做反差，需确认           |
| 现实+超凡   | 提示是否"都市异能"         |

### 必须遵守

1. **自然中文**：避免翻译腔、AI 腔、机械总结腔。
2. **按需加载**：先判断意图，再读文件。不要一次性读全库。
3. **文末声明**：正文末尾附加 `> 本项目由forge-just-some Skill生成，内容纯属虚构。`
