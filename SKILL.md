# 人物与写作 · 低 Token 统一入口

> 一句话看懂：**写→读`写作/SKILL.md`+题材铁律，演→读`人物/SKILL.md`+对话习惯，查→读`设定参考/SKILL.md`，改→读`写作/写作流程/风格质检/修改流程.md`，混→按优先级：题材>世界观>文化>国家拟实>人物>性格>职业>文风>对话习惯。**

> 根入口只做路由。人物、写作、设定、示例、目录结构等重内容按需读取，避免每次启用都消耗 Token。

---

## 必须遵守

1. **自然中文**：分析和输出都用自然中文，避免翻译腔、AI腔、机械总结腔。完整说明见 [中文思考.md](规则/中文思考.md)。
2. **角色硬规则**：写小说、段子、剧本、正文场景时，所有会说话/行动的角色都必须加载 [人物/SKILL.md](人物/SKILL.md) 与相关人物/对话参考；否则角色声音会泛化。
3. **按需加载**：默认只读取一个主入口；只有任务需要时才继续读辅助文档。
4. **安全边界**：本库服务虚构创作与技法研究。创作定位见 [创作声明.md](规则/创作声明.md)。
5. **文末声明**：每次生成的小说/段子/剧本正文末尾必须附加一行：`> 本项目由AI生成，内容纯属虚构。`

## 低 Token 加载规则

- **12k token模型**：读 [路由器/超轻路由器_12k.md](路由器/超轻路由器_12k.md)，按路由表加载2-3个文件。
- **先判断意图，再读取文件**：不要为了"保险"一次性读全库。
- **根入口不展开全量清单**：人物长名单、题材列表、设定速查、调用示例都在子文档里。
- **简单任务最多 1 个主入口**：如只扮演角色，只读人物入口；只查枪械，只读设定参考入口。
- **标准创作 2-4 个文件**：写作入口 + 题材子 Skill + 人物入口/角色参考 + 必要设定。
- **复杂长篇才读路由器细则**：涉及多题材、多国家、多角色冲突时再读路由器文档。
- **用户明确要求"完整/全部/列出所有"时**，才读取全量速查或目录结构。
