## 写之前必做：设定输入处理

当用户一次性给出大量设定/大纲/角色卡时，不要直接开写。**先对齐认知，再动笔。**

1. **拆解** — 按类别归类：世界观、角色、剧情线、规则系统、禁忌事项
2. **排列** — 重组成清晰结构：表格/列表/层级标题。让用户一眼看懂你理解了什么。
3. **补充** — 补上用户没提到但该有的（标注"补充"）。比如：用户给了角色性格但没给说话习惯→补上口头禅和语气。
4. **展示** — 在对话里显示整理好的版本。不要只贴回用户的原文，要展示你的理解和补充。
5. **确认** — 问用户"这样可以吗？有没有要调整的？"
6. **确认后才开写** — 先对齐 = 避免写 3000 字被推翻。

## 写作流程

**第 1 步：加载题材铁律**

```
确定题材 → 读 写作/类型/{题材}类/SKILL.md（含铁律 + 速查 + 检查清单）
多题材叠加 → 每个 SKILL 都读
```

**第 2 步：写作（遵循 `写作引擎.md` 的通用原则）**

- 场景：把读者扔进去，用感官细节锚定空间
- 角色：欲望 + 恐惧 + 矛盾，遮住名字能分辨谁在说话
- 信息：先让读者好奇再给答案，一次只给一个信息
- 节奏：句长/段落/信息密度三个杠杆控制快慢，变化本身才是节奏
- 情感：用身体反应、行为、环境三层传达，不直接陈述
- 对话：每句话至少推动剧情/揭示角色/埋下信息
- 结尾：让读者想翻下一页，不总结不升华不预告

**第 3 步：按需加载参考文件**

- 需要专业知识？→ `02_专业知识.md`
- 需要素材/场景库？→ `03_素材库.md`
- 需要写作技法？→ `05_技法研究.md`
- 需要角色？→ `人物/SKILL.md`
- 需要设定？→ `设定参考/SKILL.md`

**第 4 步：写完后质检**

读 `写作/写作流程/质检/04_写作后质检表.md` + 题材 SKILL.md 中的质检清单。

## 与人物 skill 联动

- **人物 skill** 提供角色原型（225 种）和对话习惯
- **写作 skill** 提供题材铁律和写作引擎
- 叠加使用：人物角色 + 写作题材 = 有灵魂的创作

示例：人物（病娇）+ 恐怖类 = 病娇恐怖 | 人物（社恐）+ 悬疑+恐怖 = 社恐密室逃脱
