# 数字化递进公式系统 v1.0

> 将"冲突升级·信息解密·情绪压强·格局递进"四个维度量化为可计算、可检验的公式。
> 用于写作过程中实时评估章节质量，发现短板后定向修改。

---

## 一、冲突升级递进

```
C_next = C * (1 + S) * (1 + R)
```

| 参数 | 含义 | 范围 | 说明 |
|------|------|------|------|
| C | 当前冲突强度 | 1-100 | 1=日常摩擦，100=生死存亡 |
| S | 赌注系数 | 0.1-2.0 | 每增加一层利害关系+0.3 |
| R | 退路系数 | 0-1.0 | 无退路=1.0 |

**赌注系数 S 规则：**

| 利害关系 | S值 |
|---------|-----|
| 个人利益受损 | 0.2 |
| 队友/NPC利益受损 | 0.3 |
| 团队/势力存亡 | 0.5 |
| 世界/文明存亡 | 0.8 |
| 叠加上限 | 2.0 |

**退路系数 R 规则：**

| 退路状态 | R值 |
|---------|-----|
| 有明确逃生方案 | 0.1 |
| 逃生方案存在但代价大 | 0.3 |
| 需牺牲他人 | 0.5 |
| 逃生方案已失效 | 0.8 |
| 无任何退路 | 1.0 |

**铁律：** C_next >= C * 1.3，否则冲突停滞。必须在本章内增加一个"拧紧"事件。

## 二、信息解密递进

```
P = (K * U) / D
```

| 参数 | 含义 | 范围 | 说明 |
|------|------|------|------|
| P | 悬念压强 | 0-100 | 越高读者越急迫 |
| K | 已知信息量 | 0-100 | 读者掌握的关键信息点数 |
| U | 未知威胁感 | 1-10 | 未解之谜带来的压迫感 |
| D | 解答速度 | 1-10 | 每多少千字给出一个关键解答 |

**未知威胁感 U 映射：**

| U值 | 感受 |
|-----|------|
| 1 | 知道有问题，但不觉得致命 |
| 4 | 知道不解决会出事，但还有时间 |
| 7 | 不解决会死人或陷入绝境 |
| 9 | 不解决会彻底崩坏，且倒计时明显 |

**最优区间：**
- D 最优：4-6（每4-6千字一个解答）
- P 最优：40-70

**信息增量公式：**
```
K_next = K + delta_K - delta_A
```
- delta_K = 本章新揭示的信息量
- delta_A = 本章作废/推翻的旧信息量
- 铁律：delta_K 应始终大于 delta_A，保持净增加

**调节规则：**
- P < 30 → 增加U或降低D（增加威胁感或加快揭露节奏）
- P > 80 → 下一章前给出部分解答（避免读者焦虑失控）

## 三、情绪压强递进

```
E = B * (1 + V) * (T / T_max)
```

| 参数 | 含义 | 范围 | 说明 |
|------|------|------|------|
| E | 情绪压强值 | 0-100 | 读者紧张/压迫程度 |
| B | 基线紧张度 | 0-100 | 由场景决定 |
| V | 节奏变异系数 | 0-1.0 | 快慢交替带来的额外压迫 |
| T | 距上次喘息的字数 | 千字 | 最长不超过T_max |
| T_max | 读者承受阈值 | 千字 | 按类型设定 |

**基线紧张度 B 映射：**

| 场景 | B值 |
|------|-----|
| 安全区 | 10 |
| 日常活动 | 20 |
| 探索未知 | 40 |
| 战斗/追逐 | 60 |
| 生死逃亡 | 80 |
| 绝望/必死 | 95 |

**节奏变异系数 V 映射：**

| V值 | 节奏模式 |
|-----|---------|
| 0.0 | 匀速推进 |
| 0.3 | 快-慢交替一次 |
| 0.6 | 快-慢-快-极快多次变化 |
| 0.8 | 慢节奏中插入突然紧张点 |

**各类型读者承受阈值 T_max：**

| 类型 | T_max（千字） |
|------|-------------|
| 规则怪谈 | 8 |
| 末世生存 | 10 |
| 悬疑推理 | 6 |
| 玄幻升级 | 12 |
| 都市日常 | 15 |

**铁律：**
- 当 T > T_max 时，必须插入喘息点（E下降至少30%）
- 喘息点形式：日常对话、安全区、回忆插叙
- 目标波动率：(E_max - E_min) / E_avg >= 1.5

## 四、格局递进

```
G = R * (1 + sum(I_i))
```

| 参数 | 含义 | 范围 | 说明 |
|------|------|------|------|
| G | 当前格局值 | 1-1000 | 每扩大一个数量级约乘10 |
| R | 初始影响半径 | 1-100 | 1=单人房间，100=一个城区 |
| I_i | 事件溢出因子 | 0.1-2.0 | 每个导致影响力外溢的事件 |

**影响半径 R 映射：**

| R值 | 空间范围 |
|-----|---------|
| 1-5 | 单人空间（病房、卧室、囚室） |
| 6-20 | 小型建筑（一栋楼、一个据点） |
| 21-50 | 中型区域（一个社区、一条街区） |
| 51-100 | 大型区域（一个城区、一座城市） |

**事件溢出因子 I 映射：**

| 事件类型 | I值 |
|---------|-----|
| 事件被目击 | 0.2 |
| 事件结果被传播（消息/谣言） | 0.3 |
| 事件引起他人主动行动 | 0.5 |
| 事件改变势力平衡 | 1.0 |
| 事件引起更高层存在关注 | 2.0 |

**铁律：** 每3-5章，G应至少翻倍，否则格局停滞。必须触发格局扩大事件。

## 五、综合递进评分（单章质量）

```
章节评分 = (delta_C + delta_P + delta_E + delta_G) / 4
```

| 参数 | 含义 | 归一化方式 |
|------|------|-----------|
| delta_C | 冲突强度章节增长率 | 归一化到0-100 |
| delta_P | 悬念压强净变化 | 绝对值，不超过20 |
| delta_E | 情绪压强波动幅度 | 理想15-30 |
| delta_G | 格局增长率 | 归一化到0-100 |

| 评分 | 等级 | 动作 |
|------|------|------|
| >= 80 | 高质量递进章节 | 通过 |
| 60-79 | 合格 | 可用，有优化空间 |
| 40-59 | 有短板 | 需修改对应维度 |
| < 40 | 不合格 | 重写 |

## 六、铁律速查（可直接写入skill约束）

```
【冲突升级铁律】
计算每章的C值。若C_next < C * 1.3，则自动增加一个冲突拧紧事件
（如关闭退路或提高赌注）。

【信息解密铁律】
维持P在40–70之间。若P < 40，则在本章内至少增加一个未解答的新信息点
（delta_K >= 3）。

【情绪压强铁律】
当连续超过T_max字数未插入喘息点时，强制插入一个E下降30%以上的场景。

【格局递进铁律】
每完成一个剧情单元（3-5章），G必须至少翻倍，否则触发格局扩大事件。
```

## 七、示例：评估《好像理性过头了》第一章

```python
example = {
    "conflict_growth": 73,      # 冲突从15到100，增长率73（归一化）
    "info_pressure_change": 22, # P从21到43
    "emotion_swing": 16,        # E从42到58
    "scale_growth": 119,        # G从37.5到82，归一化上限100
}
# 综合评分 = (73 + 22 + 16 + 100) / 4 = 52.75 → 有短板，需修改
# 短板分析：情绪压强波动太小（16），需要增加情绪起伏
```
