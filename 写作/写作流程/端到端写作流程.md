# 端到端写作流程

> 从需求到归档的完整写作工作流。每个Phase列出具体动作和对应文件路径。

---

## Phase 1: 需求确认

```
□ 确定题材（可叠加，如恐怖+亲密）
  → 参考: 写作/SKILL.md 检查表选择题材
□ 确定篇幅（短篇/中篇/连载章节）
□ 确定文风（第三人称/第一人称/起点风/晋江风/轻小说风/剧本风）
  → 参考: 写作/SKILL.md 文风速查表
□ 确定出场角色及关系
  → 角色卡路径: 人物设定/{角色}.md
  → 角色速查: 人物/references/research/速查表.md
□ 确定结构模板（三幕式/英雄之旅/爽文/推理/无限流）
  → 参考: 写作/SKILL.md 剧情结构速查
```

---

## Phase 2: 加载

```
□ 加载题材 SKILL.md
  → 路径: 写作/类型/{题材}类/SKILL.md
  → 多题材叠加时全部加载
□ 加载铁律与核心规则
  → 路径: 写作/类型/{题材}类/SKILL.md（铁律已含在内）
□ 加载角色档案（性格标签、当前状态、关系值）
  → 路径: 人物/SKILL.md → 人物设定/{角色}.md
□ 加载对话习惯
  → 必读: 人物/references/research/对话习惯/核心/00_核心理论.md
  → 速查: 人物/references/research/对话习惯/对话习惯_速查.md
□ 加载世界观/设定（如有长篇总谱）
  → 路径: 小说总谱/世界观档案.md
□ 加载时间线与伏笔表
  → 路径: 小说总谱/时间线.md、小说总谱/伏笔追踪.md
```

---

## Phase 3: 规划

```
□ 选定结构模板，确定本章在整体中的位置
  → 模板库: 写作/类型/{题材}类/04_写作模板/
□ 规划场景链（本章包含哪些场景，顺序与衔接）
□ 规划角色压力点（本章推动哪些心理变化）
□ 规划伏笔动作（埋设/维持/回收）
  → 更新: 小说总谱/伏笔追踪.md
□ 确定本章结尾类型（悬念钩/情感落点/信息释放）
□ 执行写作前检查表
  → 参考: 写作/写作流程/写作前检查表.md
  → 输出确认摘要（题材、角色状态、时间点、伏笔、文风、技法）
```

---

## Phase 4: 写作

```
□ 遵循题材铁律
  → 来源: 写作/类型/{题材}类/SKILL.md（含铁律） 全部文件
□ 应用题材技法
  → 参考: 写作/类型/{题材}类/05_技法研究/
□ 调用素材库（按需）
  → 参考: 写作/类型/{题材}类/03_素材库/
□ 用角色声音写对话
  → 对照角色卡的口头禅、语气词、方言
  → 每段对话自测: 遮住名字能否分辨角色？
□ 控制节奏: 长短句交替，铺垫→高潮→喘息
□ 注意去AI味: 避免"不禁/宛如/仿佛"堆砌，避免"心中暗想"
```

---

## Phase 5: 质检

```
□ 执行写作后质检表
  → 参考: 写作/写作流程/写作后质检表.md
□ 第一层基础质检: 角色一致性、对话辨识度、文风统一、世界观、时间线、伏笔
□ 第二层题材专项: 按题材类型逐项检查（恐怖递进/亲密因果链/战斗后果链等）
□ 第三层去AI味: 检测AI腔、翻译腔、信息重复
□ 第四层结构节奏: 开头钩子、结尾悬念、中段是否有水、节奏是否有张弛
□ 输出质检报告（通过/警告/失败 + 修复项）
```

---

## Phase 6: 归档

```
□ 保存成稿
  → 路径: outputs/{小说名}/{章节}.md
□ 更新小说总谱（如有）
  → 更新: 小说总谱/时间线.md（本章时间点）
  → 更新: 小说总谱/伏笔追踪.md（新埋/已回收的伏笔）
  → 更新: 小说总谱/角色状态.md（角色状态变化）
□ 记录本章关键信息供下一章衔接
  → 末尾场景、角色位置、情绪状态、未解悬念
```

---
