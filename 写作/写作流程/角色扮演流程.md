# 互动式角色扮演流程

> 用人物与写作skill系统做互动式角色扮演。
> 相比传统角色卡，本系统提供225种角色+对话习惯+神态描写+五感词库+冲突递进，角色深度远超几百字的角色描述。

---

## 角色扮演模式启动流程

### 第零步：创建角色扮演文件夹

用户确认角色扮演设定后，自动创建文件夹：

```
角色扮演/[故事背景命名]/
├── 角色卡/
│   └── [用户角色名].md     ← 用户的角色卡
│   └── [AI角色名].md       ← AI扮演的角色卡（如有）
├── 记忆卡/
│   └── 记忆卡.md           ← 每5-10轮对话后更新
├── 世界观/
│   └── 设定.md             ← 世界观设定
├── 对话记录/
│   └── [日期].md           ← 每天/每场一个对话记录文件
└── 地图/
    ├── config.json         ← 地图配置数据
    └── 地图编辑器.html      ← 可视化地图（复制自工具/地图编辑器/）
```

**文件夹命名规则：** 结合故事背景+题材关键词，如"废土猎奇""都市禁忌""校园暗黑""末日求生"等。

### 第一步：加载设定

用户说"我要扮演XX"或"开始角色扮演"时，自动加载：

1. **写作入口** → `写作/SKILL.md`（写作质量总控）
2. **题材Skill** → `写作/类型/{题材}类/SKILL.md`（题材铁律+核心张力）
3. **去AI味清单** → `写作/写作流程/风格质检/去AI味清单.md`（禁用句式+替换方案）
4. **角色卡** → `人物设定/*.md` 或 `人物/SKILL.md` 中的角色速查
5. **世界观** → 对应的 `设定参考/世界观/[类型]/世界规则.md`
6. **对话习惯** → `人物/references/research/对话习惯/核心/00_核心理论.md`
7. **记忆卡** → `记忆卡.md`（如果是继续扮演）

**世界观加载规则（优先级从高到低）：**
1. 用户指定了世界观 → 加载对应的 `设定参考/世界观/[类型]/世界规则.md`
2. 都没有 → 加载 `设定参考/世界观/现代都市/` 或按题材自动匹配

### 第1.5步：生成地图

根据世界观设定生成初始地图：

1. 复制 `工具/地图编辑器/地图编辑器.html` → `角色扮演/[项目名]/地图/地图编辑器.html`
2. 从世界观中提取已知地点 → 生成 `地图/config.json`
3. 角色初始位置标记为当前位置（discovered: true）
4. 角色已知但未到达的地点标记为迷雾（discovered: false）

**地图分类预设**（按题材自动选择）：
- 废土末日：安全区(绿)/危险区(红)/资源点(黄)/辐射区(紫)/未知(灰)
- 都市校园：居住区(粉蓝)/教学工作(黄)/商业(橙)/公共(绿)/特殊(紫)
- 地下迷宫：已探索(亮)/半探索(暗)/未探索(黑+???)/BOSS(红)/宝箱(金)

详细规则见 `工具/地图编辑器/使用指南.md`

### 第二步：确认角色扮演模式

告诉Agent以下规则：

```
你现在进入角色扮演模式。规则：
1. 你扮演[角色名]，用户扮演[用户角色]
2. 每次回复300-800字，包含：场景描写+角色行动+对话+感官细节
3. 对话必须符合角色的说话习惯（参考角色卡）
4. 场景描写必须包含至少2种感官（视觉/听觉/嗅觉/触觉/味觉）
5. 情感不能直接陈述，用身体反应和行为表达
6. 禁用词：仿佛/似乎/不禁/竟然/居然/一时间
7. 每次回复结尾留一个钩子或选择点，让用户决定下一步
8. 用户的行动会影响剧情走向，不是固定剧本
```

### 第三步：互动式写作

每次用户输入后，Agent的回复结构：

```
[场景描写] — 环境/氛围/感官（50-100字）
[角色行动] — AI角色在做什么（100-200字）
[对话] — 符合角色说话习惯的对话（100-200字）
[感官/心理] — 身体反应/情绪暗示（50-100字）
[钩子/选择点] — 留给用户的决策空间（30-50字）
```

### 第四步：记忆卡更新

每5-10轮对话后，自动更新记忆卡：

**记忆卡模板：**

```markdown
# [小说名/角色名] · 角色扮演记忆卡

> 每5-10轮对话后更新。

---

## 当前进度
- 已进行轮数：N轮
- 当前场景：[地点/时间/氛围]
- 当前阶段：[初遇/暧昧/冲突/深入/高潮/余韵]

## 已发生的关键事件

| 轮次 | 事件 | 角色关系变化 |
|------|------|------------|
| 第X轮 | [事件摘要] | [关系变化] |

## 角色当前状态

| 角色 | 情感状态 | 身体状态 | 位置 |
|------|---------|---------|------|
| [AI角色] | [当前情感] | [身体状况] | [在哪] |
| [用户角色] | [当前情感] | [身体状况] | [在哪] |

## 未解决的伏笔/悬念

| 伏笔 | 埋设轮次 | 状态 |
|------|---------|------|
| — | — | — |

## 下一步可能走向

- 选项A：[描述]
- 选项B：[描述]
- 选项C：[描述]
```

### 第五步：地图更新

对话过程中，以下事件触发地图更新：

| 事件 | 地图操作 | 示例 |
|------|----------|------|
| 角色到达新地点 | `discoverLocation(id)` 解除迷雾 | 走到加油站→加油站从迷雾变为可见 |
| 角色得知新地点 | `addLocation(tab, data)` + discovered: false | NPC说"北边有医院"→地图新增迷雾标记 |
| 地点被摧毁/封锁 | 删除或标记 | 加油站被炸→从地图移除 |
| 新区域解锁 | 新增tab或建筑 | 发现地下设施→新增"地下层"标签页 |
| 角色位置变化 | 更新当前位置标记 | 从加油站走到医院→当前位置切换 |

详细规则见 `工具/地图编辑器/使用指南.md`

## 对话记录格式

每轮对话记录到 `对话记录/[日期].md`：

```markdown
# [项目名] · 对话记录 · [日期]

---

## 第1轮

**用户**：
> [用户的输入原文]

[Agent的叙事正文]

**状态变化**：[状态变化用粗体标记]

---

## 第2轮

**用户**：
> [用户的输入原文]

[Agent的叙事正文]

**状态变化**：[如有]

---

**📍 第2轮结束 · 当前状态**：

| 项目 | 状态 |
|------|------|
| 位置 | ... |
| 角色状态 | ... |
| 场景 | ... |
| 手中物品 | ... |

**第3轮选项**：
- A. ...
- B. ...
```

**格式铁律**：
- 用户输入用 `>` 引用块
- 叙事正文用**普通段落**
- 每轮末尾附**状态表 + 下一轮选项**
- 用户的输入**原样记录**，不改一个字
- Agent的输出**原样记录**，这是对话历史
- 状态变化只记**有变化的**，没变化就不写

---

## 用户输入类型处理

| 用户输入 | 类型 | Agent处理方式 |
|----------|------|------------|
| "你是什么人？" | 对话 | NPC回应对话 |
| "我拔出枪对准他" | 动作 | 描写场景变化+NPC反应 |
| "三天后，我们到了营地" | 旁白/时间跳跃 | 接受用户的时间控制，描写新场景 |
| "我选A，跟他走" | 决策 | 推进剧情，记录到记忆卡 |
| "跳过这段" | 跳过 | 快进到下一个关键节点 |

**铁律**：用户的输入是**指令**，不是建议。用户说跳过就跳过，用户说时间跳跃就跳跃，用户说角色做了什么就做了什么。Agent只负责NPC的反应和场景描写，不替用户决定用户角色的行为。

---

## Agent输出结构

每轮输出包含**三个层面**（可以合并写，但三个维度都要有）：

```
[氛围层] 环境/感官/气氛，20-50字（可选，不是每轮都需要）

[NPC层] NPC的反应/对话/动作，100-300字（核心）

[场景层] 场景变化/新信息/触发点，50-100字（可选）
```

**示例**：
> （氛围）加油站里安静得能听到灰尘落地的声音。
> 
> （NPC）他往后退了一步，后背撞上货架。罐头哗啦掉了一地。
> "别……别开枪。"他举起双手，那只黑色的手在微微发抖。"我不是掠夺者。我只是来找药的。"
> 
> （场景）他的眼睛盯着你的枪口，但你注意到他的视线偶尔往下瞟——看你腰间的医疗包。

## NPC管理

### 动态NPC出场

不在角色卡里的NPC → 第一次出场时由Agent快速创建 → 记到记忆卡 → 如果戏份多了再单独建角色卡

**快速NPC模板**（记在记忆卡里）：
```
[名] | [身份] | [一句话特征] | [与主角关系] | [当前状态]
```

### NPC对话辨识度铁律

每个NPC说话必须有辨识度——遮住名字能分出是谁在说。

**检查**：
- NPC的用词符合身份吗？（废墟拾荒者不会说成语）
- NPC的语气跟性格一致吗？（冷酷的不会突然撒娇）
- NPC之间说话能区分吗？

---

## 角色扮演专用的质量约束

| 维度 | 约束 |
|------|------|
| 角色一致性 | 每次回复前检查：角色的说话方式是否符合角色卡？ |
| 场景连贯性 | 上一轮在卧室，这一轮不能突然到操场（除非有过渡） |
| 情感递进 | 不能一轮还是陌生人，下一轮就深爱（要有铺垫） |
| 用户能动性 | 用户的选择必须影响剧情，不能强行拉回固定路线 |
| 禁忌底线 | 角色的底线不能被随意突破（除非有足够铺垫） |

---

## 与"继续创作"的区别

| 模式 | 特点 | 适用场景 |
|------|------|---------|
| 继续创作 | Agent写完整章节，用户审核修改 | 写小说 |
| 角色扮演 | 用户和Agent轮流推进剧情，互动式 | 玩角色扮演/探索剧情 |

两种模式可以切换：
- 角色扮演玩到满意的剧情 → "把这段写成正式章节" → 切换到继续创作模式
- 继续创作写到某个场景 → "我想亲自体验这段" → 切换到角色扮演模式
