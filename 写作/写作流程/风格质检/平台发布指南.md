# 平台发布指南

> 同一本小说，在起点和在番茄的写法可能完全不同。平台不是"写完再选"的附属品，而是从第一章就要考虑的战略决策。选错平台，再好的故事也可能扑街。

---

## 一、为什么需要了解平台差异

很多新手作者觉得"我把小说写好就行了，发哪个平台无所谓"。这是一个致命的误解。

**同一个故事，不同平台的写法差异：**

| 维度 | 起点中文网 | 番茄小说 | 晋江文学城 |
|------|-----------|---------|-----------|
| 开头 | 慢热，用3-5章建立世界观 | 3句话必须有钩子 | 氛围感开场，情绪先行 |
| 节奏 | 每30章一个大高潮 | 每章一个爽点 | 每2-3章一个情感转折 |
| 字数 | 3000-5000字/章 | 1500-3000字/章 | 2000-4000字/章 |
| 付费点 | 第30章左右上架 | 全文免费 | 第V章（看收藏数） |
| 读者耐心 | 较高，愿意等铺垫 | 极低，3秒定生死 | 中等，但要求情感质量 |

**平台选择本质上是选择：**
- 你的读者是谁（年龄、性别、阅读习惯）
- 他们愿意为什么付费（订阅、打赏、看广告）
- 你的作品怎样被发现（编辑推荐、算法推荐、榜单）
- 你该用什么节奏讲故事（快、慢、细腻）

不理解这些差异就开书，等于闭着眼睛开车。

## 二、主要平台特点对比

### 2.1 起点中文网

**读者群体：** 25-35岁男性为主，阅读经验丰富，口味挑剔。很多人看了十年以上网文，套路见得太多，一般的桥段根本打动不了他们。

**付费模式：** 订阅制，按章付费。前30章左右免费（公众期），之后进入VIP上架，读者需要付费才能继续阅读。这是网文最传统的付费模式，读者的付费意愿和能力都比较强。

**推荐机制：** 编辑推荐 + 数据推荐双轨制。编辑会看你的书有没有潜力，数据（收藏、推荐票、点击）好的书会获得更多曝光。新书期有新书榜，上架后有月票榜。

**风格偏好：**
- 文笔不能太差，至少要通顺流畅、没有明显语病
- 设定要严谨，世界观经得起推敲——起点读者会考据
- 节奏相对慢，允许铺垫，但铺垫必须有信息量
- 主角要有成长线，不能一上来就无敌

**字数要求：** 每章3000-5000字。低于3000字读者会觉得"水"，高于5000字如果信息密度不够也会被骂。

**注意事项：**
- 不能太套路，起点读者见过太多"系统流""穿越流"的模板了
- 不能太文青，起点毕竟是商业网文平台，纯文学写法没人看
- 不能开头就铺设定——用故事带设定，别用设定讲故事
- 更新要稳定，日更是基本要求，万字更新的作者有加成

---

### 2.2 晋江文学城

**读者群体：** 20-30岁女性为主，也有相当数量的男性读者和年龄更大的女性读者。晋江读者对文笔的要求是所有平台里最高的。

**付费模式：** 订阅制 + 打赏。晋江有自己的虚拟货币系统，读者购买后按章订阅。打赏（营养液、地雷等）也是重要的收入来源。热门作品的打赏收入有时甚至超过订阅。

**推荐机制：** 榜单驱动。晋江有大量榜单——新文榜、收藏榜、积分榜、推荐榜、月榜、季榜、年榜等。收藏数和评论数是上榜单的核心指标。编辑也会推荐，但读者自发的榜单行为更重要。

**风格偏好：**
- 文笔细腻是基本门槛，粗糙的文字在晋江活不下去
- 感情描写要好，无论是BG、BL还是GL，感情线必须是核心
- 人设立体，不能脸谱化——晋江读者会写几千字的"人设分析"
- 对话要有个性，不能所有人说话一个腔调

**字数要求：** 每章2000-4000字。晋江的章节普遍比起点短，但信息密度高。

**注意事项：**
- 不要用"起点风"写晋江文——这是最常见的错误。起点风的特点是设定宏大、打斗详细、升级明确，这些在晋江不受欢迎
- 晋江有自己的"圈子文化"，了解读者在讨论什么、喜欢什么梗很重要
- 评论区是重要的互动场所，作者回复评论会被读者记住
- 注意晋江的审核规则，某些内容有红线

---

### 2.3 番茄小说

**读者群体：** 18-40岁，覆盖面最广，但以"下沉市场"为主——也就是非传统网文读者。很多人是第一次接触网文，或者只是随便刷刷手机消磨时间。

**付费模式：** 全免费 + 广告分成。读者不需要付费，靠看广告解锁章节或者在阅读间隙插入广告。作者的收入来自平台根据阅读量、完读率等数据计算的广告分成。

**推荐机制：** 纯算法推荐。没有编辑人工筛选，完全靠数据说话。核心指标包括：完读率（多少人看完了这一章）、停留时长（读者在这一章花了多少时间）、互动率（点赞、评论、分享）。算法会把你的书推给一小批人，如果数据好就推更多人，数据差就停止推荐。

**风格偏好：**
- 节奏要快，不能有任何"废笔"——每一段都要有信息量或爽点
- 章末必须有钩子，让读者忍不住翻下一章
- 开头3章定生死——如果前3章数据不好，算法就不会再推荐了
- 语言直白，不要用复杂的修辞和隐喻
- 情绪要外露——"他很愤怒"比"他的手指微微收紧"更好（番茄读者没时间品味潜台词）

**字数要求：** 每章1500-3000字。番茄的短章模式是核心特征，读者习惯快速阅读。

**注意事项：**
- 番茄的竞争极其激烈，每天有上万本新书上线
- 算法只认数据不认人，不管你写了多久、有多少粉丝
- 前3章的数据决定了整本书的命运——投入最多精力在前3章
- 不要写慢热型故事，番茄没有"熬过前期就好了"的机会

---

### 2.4 七猫小说

**读者群体：** 30-50岁为主，比番茄的读者年龄更大，很多是从传统媒体（电视、报纸）转向手机阅读的群体。阅读经验较少，对网文套路不熟悉。

**付费模式：** 全免费 + 广告，和番茄类似。

**推荐机制：** 算法推荐，和番茄的逻辑基本一致。

**风格偏好：**
- 通俗易懂是第一要求——七猫读者的文化水平分布比起点和晋江更广
- 爽点密集且直接，不需要铺垫就能感受到爽
- 不要文艺腔，不要意识流，不要留白——直接写出来
- 人物关系要简单明了，不要搞太复杂的多线叙事

**字数要求：** 每章1500-2500字。比番茄更短，因为七猫读者更没耐心。

**注意事项：**
- 七猫的审核比番茄宽松，但不代表可以写低质量内容
- 七猫用户对"套路"的容忍度比起点高——他们觉得好看的套路就是好故事
- 适合写"爽文""甜文""打脸文"这种情绪驱动型作品
- 标题和封面极其重要，因为七猫用户的决策时间更短

---

### 2.5 书旗小说

**读者群体：** 25-40岁，介于起点和番茄之间。既有传统网文读者，也有新入门的读者。

**付费模式：** 混合模式——部分作品免费（广告分成），部分作品付费订阅。作者可以选择自己的作品走哪条路线。

**推荐机制：** 算法推荐 + 编辑推荐混合。

**风格偏好：**
- 跟番茄类似但节奏可以稍微慢一点
- 允许一定程度的铺垫，但不能太长
- 质量要求比番茄高，但比起点低

**字数要求：** 每章2000-3500字。

**注意事项：**
- 书旗的流量不如番茄和七猫，但竞争也相对小
- 适合作为多平台分发的补充平台
- 书旗的读者对"品质"有一定追求，纯套路文不太吃得开

## 三、平台选择指南

### 新手第一本书选哪里？

| 你的情况 | 推荐平台 | 原因 |
|---------|---------|------|
| 第一次写网文，想试试水 | 番茄 | 门槛低、反馈快、不需编辑审核 |
| 想赚订阅费，愿意慢慢积累 | 起点 | 读者付费意愿最强，收入天花板高 |
| 写女频（言情、耽美、百合） | 晋江 | 女频读者最集中的平台，生态成熟 |
| 想要最大流量 | 番茄/七猫 | 免费模式读者基数大，算法推爆款 |
| 写严肃向、文学性强的作品 | 起点/晋江 | 这两个平台的读者最能接受高品质作品 |

### 多平台分发策略

如果你精力足够，可以考虑多平台分发：

**能同时发的：** 番茄 + 七猫 + 书旗（都是免费模式，互不冲突）

**不能同时发的：** 起点和晋江通常有独家协议——签约后不能在其他平台发同一部作品

**分发原则：**
1. 先在一个平台证明作品有数据，再分发到其他平台
2. 不同平台可以用同一个故事，但标题、简介、标签要针对平台调整
3. 不要把同一本书同时发到起点和番茄——写作风格完全不同，强行兼顾会两头不讨好

## 四、各平台的写作技巧差异

### 4.1 起点：深度为王

**核心策略：** 用设定深度和世界观严谨性留住读者。

- **设定要有"可探索性"**——读者看你的书不只是看故事，还想知道"这个世界还有多少秘密"
- **主角成长线要清晰**——每30章应该有一个明确的能力/地位/认知的提升
- **配角要有自己的故事**——起点读者会记住配角，如果配角只是工具人会被骂
- **伏笔回收**——前面埋的线后面必须收，起点读者会翻回去查

**节奏模板：**
```
第1-5章：建立主角身份 + 第一个小事件
第6-15章：展开世界观 + 第一次能力展示
第16-30章：第一个大冲突（公众期高潮）
第31章：上架，必须是读者最想看下一章的时候
```

### 4.2 番茄：钩子为王

**核心策略：** 每一章都是一个"广告"，目的是让读者看下一章。

- **开头3章必须包含：** 主角是谁、面临什么困境、第一个爽点在哪里
- **每章结尾必须有钩子**——悬念、反转、冲突升级，让读者停不下来
- **爽点间隔不超过1000字**——如果一段文字里没有"爽"或"悬念"，就要删掉或改写
- **情绪要直接**——不用暗示和留白，直接告诉读者"他很爽""她很震惊"

**节奏模板：**
```
第1章：主角出场 + 遭遇困境 + 获得金手指（3000字内完成）
第2章：第一次使用金手指 + 打脸配角
第3章：更大的冲突引出 + 第一个悬念钩子
之后：每章一个小爽点，每3-5章一个大爽点
```

### 4.3 晋江：情感为王

**核心策略：** 用细腻的感情描写和立体的人设留住读者。

- **感情线是核心**——无论是言情、耽美还是百合，读者来看的是"这两个人怎样在一起"
- **人设要有反差萌**——表面冷酷内心温柔、看似精明实则天然，这种反差是晋江读者最爱的
- **对话要出彩**——晋江读者会截图分享"名台词"，好的对话是传播利器
- **氛围感很重要**——雨天、深夜、空旷的教室，场景本身要有情绪

**节奏模板：**
```
第1-3章：主角出场 + 另一位主角出场 + 第一次相遇/互动
第4-10章：关系建立，日常互动，埋感情线
第11-20章：第一个情感转折（误会/分离/危机）
第21章起：感情升温 + 主线推进
V章（上架）：选在感情线最关键的地方
```

### 4.4 七猫：爽感为王

**核心策略：** 用最直接的方式制造爽感。

- **每章必须有一个"爽点"**——打脸、逆袭、获宝、升级，总之要让读者觉得"爽"
- **反派要脸谱化**——不要给反派太多深度，七猫读者要的是"坏人被打脸"的快感
- **语言要通俗**——初中文化水平就能无障碍阅读
- **金手指要明显**——读者要能一眼看出主角"牛在哪里"

## 五、发布前准备

### 5.1 书名设计

**基本原则：** 短（2-8字）、有辨识度、暗示题材。

| 题材 | 好书名 | 差书名 |
|------|-------|--------|
| 都市重生 | 重生2005 | 我的重生之路与人生感悟 |
| 修仙 | 万古第一神 | 异世界修仙传奇故事 |
| 系统流 | 全球降临 | 获得系统后的我开始了新生活 |
| 言情 | 他的白月光 | 总裁与灰姑娘的浪漫爱情 |
| 悬疑 | 第七个证人 | 关于一起离奇案件的调查 |

**起名技巧：**
- 用数字（三个、第七、万古）增加记忆点
- 用反差词（温柔的杀意、甜蜜的陷阱）制造冲突感
- 用疑问句（谁杀了她）制造悬念
- 不要用生僻字——读者搜索不到就找不到你的书

### 5.2 简介设计

**黄金公式：** 3句话说清卖点 + 1个钩子。

```
第1句：主角是谁 + 面临什么处境
第2句：核心设定/金手指是什么
第3句：主角要做什么（目标/冲突）
钩子句：一个让人想点进去的问题或反转
```

**示例：**
> 林远是个普通程序员，996加班到凌晨三点猝死在工位上。（主角+处境）
> 再睁眼，他回到了大学毕业那一年，脑子里多了一个能预测股市走向的系统。（设定）
> 这一世，他不想再当牛马。（目标）
> 但系统告诉他：你只有三年的使用时间。（钩子）

**简介禁忌：**
- 不要写世界观设定——简介不是说明书
- 不要超过150字——太长没人看
- 不要用"且看主角如何XXX"这种老套结尾
- 不要在简介里剧透关键转折

### 5.3 封面与标签

**封面选择：**
- 跟题材匹配——修仙就用仙侠风，都市就用现代风
- 有辨识度——在一堆封面缩略图中能被看到
- 颜色对比强烈——冷色调（蓝紫）适合悬疑/科幻，暖色调（红橙）适合热血/言情
- 不要用AI生成的模糊人脸当封面——番茄上这类封面泛滥，毫无辨识度

**标签选择——比你想象的重要10倍：**
- 标签决定了算法把你的书推给谁
- 选错标签 = 推给错误的读者 = 数据差 = 算法停止推荐
- 主标签决定大方向，副标签决定精准人群
- 建议研究同类型热门书的标签组合

### 5.4 分类选择

```
男频：
├── 玄幻（最大类，竞争最激烈）
├── 都市（现代背景，适合新手）
├── 科幻（小众但忠诚度高）
├── 历史（需要知识储备）
├── 仙侠（传统题材，读者稳定）
├── 游戏（系统流、数据流集中地）
└── 悬疑（近年上升趋势）

女频：
├── 言情（最大类）
├── 耽美（晋江主阵地）
├── 古言（穿越、宫斗、宅斗）
├── 现言（都市、校园、职场）
├── 幻言（修仙+言情）
└── 百合（小众但增长快）
```

## 六、更新策略

### 6.1 开书前：存稿是生存底线

**最低存稿量：** 10-20章。

存稿的意义不只是"万一有事断更"，更重要的是：
- 你可以在发布后根据读者反馈调整后续剧情
- 数据好的时候你能加更，抓住流量窗口
- 写到后面发现前面有bug，你有时间回去改

**没有存稿就开书 = 在没有降落伞的情况下跳飞机。**

### 6.2 日常更新

**基本原则：**
- 日更1-2章，每天固定时间更新
- 固定时间比每天多更重要——读者会养成习惯
- 推荐更新时间：中午12:00 或 晚上20:00-21:00（阅读高峰期）

**不同平台的更新节奏：**

| 平台 | 建议日更量 | 最低日更量 | 更新时间 |
|------|-----------|-----------|---------|
| 起点 | 4000-6000字 | 3000字 | 晚8点 |
| 番茄 | 3000-6000字 | 1500字 | 中午12点或晚8点 |
| 晋江 | 3000-5000字 | 2000字 | 晚8点 |
| 七猫 | 3000-5000字 | 1500字 | 中午12点 |

### 6.3 上架时机

**起点：** 公众期30章左右上架。上架前最好已经积累了足够多的收藏和推荐票。上架第一章必须是高潮或悬念最强的时候——让读者"不得不付费看下去"。

**番茄/七猫：** 全文免费，不存在上架。但前3章是事实上的"生死线"——数据不好就废了。

**晋江：** 看收藏数决定V章（上架章节）。通常收藏达到一定数量后开V，具体标准参考晋江的规则说明。

### 6.4 断更补救

**预防：**
- 请假比断更好——提前在作者有话说/公告里请假
- 存稿就是为了应对突发情况

**补救：**
- 恢复更新后第一天加更（至少双更）
- 在作者有话说里简单解释原因（不要长篇大论诉苦）
- 恢复更新后保持至少一周的稳定日更，重建读者信任

### 6.5 完本后

- 完本后不要立刻开新书——休息1-2周，同时在旧书的读者群里预热新书
- 新书可以在旧书的章节末尾打广告
- 最好在完本后1个月内开新书——超过3个月，读者就把你忘了
- 如果旧书成绩好，新书可以考虑同类型或同世界观——读者迁移成本低

## 七、平台发布常见错误

### 错误1：用起点风格写番茄

**症状：** 前5章在铺设定、建世界观、写主角的日常生活。番茄读者在第2章就跑了。

**改正：** 前3章必须包含金手指激活 + 第一个爽点。设定边写边带，不要集中展示。

### 错误2：用番茄风格写起点

**症状：** 节奏太快、爽点太密集、没有深度。起点读者觉得"这就是小白文"，直接弃书。

**改正：** 放慢节奏，给世界观留出展开空间。每30章一个大事件，中间用细节和设定填充。

### 错误3：书名太文艺

**症状：** 取了一个诗情画意的名字，比如"时光深处的回响"。读者根本不知道这是什么题材，不会点进来。

**改正：** 书名要让人一眼看出题材和卖点。"重生2005"比"时光深处的回响"好100倍。

### 错误4：简介太长

**症状：** 简介写了500字，从世界观设定讲到主角前世今生。读者划到第三行就关了。

**改正：** 简介控制在100-150字。3句话讲清楚核心卖点，最后一句留悬念。

### 错误5：开书前没有存稿

**症状：** 写了3章就发布，然后每天现写现发。某天有事断更，数据暴跌，算法停止推荐。

**改正：** 至少存10章再发布。这是最基础的安全垫。

### 错误6：标签选错

**症状：** 写了一本都市重生文，标签选了"玄幻""修仙"。系统把你的书推给了想看修仙的读者，他们点进来发现不是修仙，直接退出。完读率暴跌，算法判定你的书不行。

**改正：** 研究3-5本同类型热门书的标签，参考它们的标签组合。

### 错误7：更新时间不固定

**症状：** 今天中午发，明天凌晨发，后天晚上发。读者想追更但不知道什么时候能等到新章节，慢慢就不追了。

**改正：** 固定一个时间，每天准时更新。比"每天多写1000字但时间不固定"效果好得多。

### 错误8：同时在多个平台发不同作品

**症状：** 起点写一本玄幻，番茄写一本都市，晋江写一本言情。精力分散，三本都写不好，每本的数据都很差。

**改正：** 先集中精力写好一本书。等这本书完本或者稳定更新之后，再考虑开新书或转平台。一本成绩好的书比三本扑街的书有价值得多。
