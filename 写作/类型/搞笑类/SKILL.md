# 搞笑写作 Skill（速查表）

> "我以为这是个正经故事，结果笑到隔壁来敲门。"

---

## 核心张力公式

**搞笑 = 铺垫的严肃程度 x 反转的落差程度 x 反应的冷淡程度**
让读者自己绷不住笑，而不是硬塞段子。

---

## 铁律（开写前必读）

1. **笑点来自角色性格，不来自作者旁白** — 搞笑是角色的反应/行为自然产生的，不是作者在旁边吐槽。
2. **反差是最大的笑料来源** — 预期与结果的落差越大，越好笑。严肃的人做了荒唐事，或者荒唐的人突然认真。
3. **搞笑不能建立在角色的痛苦上** — 可以倒霉，但角色要自嘲而不是被嘲笑。
4. **铺垫比包袱更重要** — 好笑不是因为"说了什么"，而是因为"铺垫了多久才说出来"。
5. **预期违背比例为 7:3** — 七成符合预期，三成出乎意料。每次都猜不到=在耍人。
6. **不要解释笑点** — "哈哈好笑吧"="（这是一个谐音梗）"=笑点废了。
7. **不要为了搞笑而搞笑** — 每个笑点都要推进角色关系或剧情，否则就是注水。
8. **旁观者必须有反应** — 不能像NPC一样无视一切。吐槽/逃离/震惊/假装没看到，至少选一种。

**自检公式**：搞笑 = 铺垫的严肃程度 x 反转的落差程度 x 反应的冷淡程度

---

## 搞笑的三种底层机制

### 机制1：建立-打破
先建立一个稳定的预期，再打破它。
1. 给读者一个"正常"的场景（持续至少2-3段）
2. 在读者适应之后，扔一个不正常的东西进去
3. 不要急着处理，让角色自己面对

> 他走进会议室，PPT已经做好了，气氛严肃。他清了清嗓子。投影仪上出现的不是他的PPT，是一只猫的照片。没有人说话。他看了三秒："......这是谁的猫？"

### 机制2：信息不对称
- **读者全知型**：读者知道门后面有人，角色不知道。笑点来自角色毫无防备地走向灾难。
- **悬念型**：角色做了一件看似蠢的事，最后才揭示"原来他早就计划好了"。
- **误会型**：A以为情况是X，B以为情况是Y。读者同时知道两边，看着他们越走越远。

### 机制3：反应错位
- **反应不足**（冷处理）：事情很大，角色反应很小。"哦。"——适合冷面笑匠。
- **反应过度**（热处理）：事情很小，角色反应很大。手指被纸划了，哭得像世界末日。——适合戏精。
- **反应错位**（答非所问）：反应完全不在点上。"仓库着火了！""我上次灭火用的灭火器过期了吗？"

## 五拍结构（单章核心骨架）

| 拍 | 名称 | 占比 | 任务 |
|----|------|------|------|
| 1 | 铺垫(正常化) | 10% | 一切看起来很正常，读者放松警惕 |
| 2 | 误导(往A引) | 15% | 让读者以为接下来会发生A |
| 3 | 反转(突然去了B) | 40% | 突然。一句话就够 |
| 4 | 笑点爆发 | 20% | 不要加括号解释，给角色一个反应就好 |
| 5 | 余韵 | 15% | 角色的反思/后续影响/伏笔 |

**节奏要点**：短句=笑点，长句=铺垫，交替使用。段落长度要不规则，像心电图。

## 8种搞笑角色速查

| 角色 | 核心 | 注意 |
|------|------|------|
| 吐槽役 | 替读者说"这也太离谱了" | 不要每句都吐槽，留给最荒诞时刻 |
| 装傻役 | 做正常人绝对不会做的事 | 分真傻和装傻两种 |
| 天然呆 | 跟别人不在一个频道上 | 反应慢半拍，莫名说出厉害的话 |
| 毒舌 | 嘴太损但说得太对 | 对象最好是"活该被骂"的角色 |
| 倒霉蛋 | 喝凉水都塞牙 | 偶尔让他赢一次 |
| 戏精 | 把人生当舞台 | 平时越浮夸，认真起来越有冲击力 |
| 冷面笑匠 | 语气平，结论爆笑 | 千万不要让他自己笑 |
| 疯批 | 不受控制的变量 | 需要"安全阀"角色拉住他 |

## 角色卡速查

### 吐槽役
- 社畜型：黑眼圈快掉到下巴 | 保温杯不离手 | 衬衫永远皱的 | 口头禅"然后呢？"
- 学生型：普通到人群里找不到 | 口头禅"不，等等。"

### 装傻役
- 真天然傻：圆脸大眼睛永远在微笑 | 口头禅"诶？是这样吗？"
- 自作聪明(嘉豪)型：戴眼镜表情自信 | 口头禅"根据我的分析——"

### 旁观者反应速查

| 角色类型 | 反应方式 |
|---------|---------|
| 吐槽役 | 发消息吐槽/当面吐槽/事后复盘 |
| 老好人 | 假装没看到/主动离开/偷偷帮忙 |
| 戏精 | 加入围观/夸张反应/拍照取证 |
| 冷面笑匠 | 说一句冷笑话然后离开 |
| 社恐 | 戴耳机装没听到/发抖/原地石化 |

## 常见失败模式 Top 5

1. **硬塞笑点** → 建立角色关系，让互动产生笑点
2. **解释笑点** → 删掉所有解释，让读者自己笑
3. **反转滥用** → 7:3比例，不要每次都反转
4. **角色平面** → 角色先是人，然后才是搞笑的
5. **尴尬过度** → 到"好笑"就收，别到"社死"

## 核心规则

# 搞笑类 -- 核心规则 (铁律)

> 开写前必须阅读。违反任何一条，笑点失效。

---

## 铁律1：笑点来自角色性格，不来自作者旁白

搞笑应该是角色的反应/行为自然产生的，不是作者在旁边吐槽。

- **正例**：他一本正经地说："我已经研究过逃跑路线了。"众人看地图——他画的路线穿过了三堵墙。
- **反例**：他很蠢（作者旁白：哈哈，你看他多蠢啊！）。(作者旁白我就绷不住了)
- **违反后果**：搞笑变成尴尬，读者觉得作者在自嗨。

## 铁律2：反差是最大的笑料来源

预期与结果的落差越大，越好笑。严肃的人做了荒唐事，或者荒唐的人突然认真。

- **正例**：黑帮老大掏出手机，屏保是一只粉色小猪。
- **反例**：一个搞笑角色继续搞笑。没有反差。
- **违反后果**：笑点疲劳，读者不再觉得好笑。

## 铁律3：搞笑不能建立在角色的痛苦上（除非角色自己在笑）

可以倒霉，但角色要自嘲而不是被嘲笑。

- **正例**：他摔倒了，裤子裂了。他站起来看了看，说："至少今天风大，凉快。"
- **反例**：他摔倒了，所有人都在嘲笑他，他哭了。旁白继续嘲笑他。
- **违反后果**：从搞笑变成霸凌，读者同情角色而非觉得好笑。

## 铁律4：铺垫比包袱更重要

好笑不是因为"说了什么"，而是因为"铺垫了多久才说出来"。

- **正例**：他花了五分钟解释这个计划有多完美。然后执行的第一步就失败了。
- **反例**：他做了一件蠢事。好笑。结束。
- **违反后果**：没有铺垫的包袱不响。

## 铁律5：预期违背比例为 7:3

七成符合预期，三成出乎意料。如果每次读者都猜不到，他们不是觉得有趣，是觉得你在耍人。

## 铁律6：不要解释笑点

如果你写了一个好笑的东西，紧接着加了一句"因为xxx所以很好笑"，那这个笑点就废了。同理——

- 不要用"哈哈哈哈哈"代替写得好笑
- 不要在笑点后面加括号解释"（这是一个谐音梗）"
- 不要让角色替读者笑——"大家都笑了"不等于"这个故事好笑"

## 铁律7：不要为了搞笑而搞笑

每个笑点都要推进角色关系或剧情，否则就是注水。搞笑是手段，不是目的。

## 铁律9：旁观者必须有反应，不能像NPC

场景里有其他角色时，他们必须对正在发生的事有真实反应——吐槽、逃离、震惊、假装没看到、发消息吐槽、主动离开。不能像游戏里的NPC一样无视一切。

- **正例**：室友在卧室里戴着耳机假装没听到，但给主角发消息："你们能不能小声点。""你叫得比游戏里的Boss还惨。""我现在出门，给你们两个小时。"
- **反例**：室友在卧室里，客厅里两个人在沙发上做爱，室友毫无反应，像不存在一样。
- **违反后果**：配角变成工具人，读者出戏，场景失去真实感。

**旁观者反应速查**：

| 角色类型 | 反应方式 |
|---------|---------|
| 吐槽役 | 发消息吐槽/当面吐槽/事后复盘 |
| 老好人 | 假装没看到/主动离开/偷偷帮忙 |
| 戏精 | 加入围观/夸张反应/拍照取证 |
| 冷面笑匠 | 说一句冷笑话然后离开 |
| 社恐 | 戴耳机装没听到/发抖/原地石化 |

## 自检公式

```
搞笑 = 铺垫的严肃程度 x 反转的落差程度 x 反应的冷淡程度
```

**一句话检验**：把你写的搞笑桥段念给一个朋友听。他笑了，你赢了。他没笑，不是他不懂幽默，是你写的不够好。


## 质检清单

# 搞笑类 -- 质检清单

> 完稿前逐项检查。打勾通过，未通过则修改。

---

## 通用检查项

- [ ] 无病句和错别字
- [ ] 人称一致（无视角混乱）
- [ ] 时间线连贯（无矛盾）
- [ ] 对话自然（无AI腔/书面腔）
- [ ] 每章有明确的情节推进

## 核心规则检查

- [ ] 笑点来自角色性格/行为，不是作者旁白吐槽
- [ ] 有反差感（严肃x荒唐 / 预期x结果）
- [ ] 铺垫够长——包袱响不响取决于铺垫
- [ ] 搞笑不建立在角色的痛苦上（除非角色自嘲）
- [ ] 预期违背比例约为7:3（不是每次都反转）
- [ ] 没有"解释笑话"——如果需要解释就不好笑
- [ ] 每个笑点都推进了角色关系或剧情（不是注水）

## 角色检查

- [ ] 角色不只是搞笑——要有让人喜欢的特质
- [ ] 不看角色名字，读者能分辨谁在说话
- [ ] 搞笑方式没有破坏角色人设
- [ ] 角色之间有互动化学反应，不只是单人表演

## 节奏检查

- [ ] 节奏张弛有度（不能全篇搞笑，需要留白）
- [ ] 笑点类型有变化（不能全是一种套路）
- [ ] 短句和长句交替自然（短句=笑点，长句=铺垫）
- [ ] 段落长度不规则，像心电图而非直线
- [ ] 没有连续三个以上热笑话（太吵）或冷笑话（不好笑）
- [ ] 吐槽的时机和力度匹配荒诞程度

## 表现力检查

- [ ] 没有用"他笑了/他尴尬了"等偷懒写法（用身体语言替代）
- [ ] 感官描写在搞笑场景中被利用（视觉/听觉/触觉/嗅觉）
- [ ] 没有"哈哈哈哈哈"/"大家都笑了"等凑字数写法
- [ ] 没有网络过期用语（"家人们谁懂啊"等保质期短的表达）

## 禁忌清单

| 不要这样写 | 改成这样 |
|-----------|---------|
| 全场都笑了 | 小明把头埋进了胳膊里。肩膀在抖。旁边的人假装没看到 |
| 他觉得很好笑 | 他试图忍住。嘴唇咬得发白了。然后发出了一声很奇怪的"噗" |
| 这真是太尴尬了 | 他开始假装看手机。手机屏幕是锁着的 |
| 他的笑话不好笑 | 全场安静了三秒。有人在清嗓子。有人在假装咳嗽 |
| 她气得说不出话 | 她笑了。但那个笑容没有到达眼睛 |
| 他摔倒了很好笑 | 他以一种违反人体力学的姿势倒在了地上。旁边有人在掏手机 |

## 总分评估

每项1分，满分30分。低于20分需要大幅修改，20-25分需要局部调整，25分以上可以发布。
