# 人物Skills · 角色扮演技能库

> 详细说明见上级 `forge/SKILL.md`。

## 快速使用

1. 确定人物类型（人物类/性格类/职业类）
2. 加载 `references/research/对话习惯/核心/00_核心理论.md`
3. 加载对应的 `[类型]_skill参考.md`
4. 融合输出

### 命名规则

**默认主角名（硬性规则）：**
- **男主默认名：天蕴** | **女主默认名：邵阳**  (这是我刚写小说默认的两个主角名)

## 角色加载流程图

```
收到写作任务
    ├─ 有现成角色档案？→ 加载档案 → 跳到步骤3
    └─ 否 → 步骤1: 从速查表选标签（2-3个组合）→ 查兼容矩阵（`人物/兼容矩阵/`）
    步骤2: 加载核心理论 `references/research/对话习惯/核心/00_核心理论.md`
    步骤3: 按需加载对话参考（方言/梗/平台风格/角色特征）
    步骤4: 加载题材规则 `写作/类型/[题材]/SKILL.md`
    步骤4.5: 按需加载人物联动写作参考 `references/人物联动写作/[题材]类.md`
    步骤5: 融合输出 → 写作/扮演 → 写完更新人物档案
```

## ⚠️ 建角前必查兼容矩阵（硬性规则）

> 标签组合必须过兼容矩阵，否则角色会OOC。

查 `人物/兼容矩阵/人物兼容矩阵.md` + `性格兼容矩阵.md` + `职业兼容矩阵.md`。有冲突→调整标签或加铺垫。

## 多角色对话区分清单

1. **语气基线不同** — 每个角色默认语调必须有差异
2. **句式长度分层** — 话痨长句/话少一字/腹黑短句反问
3. **口头禅/语气词不重叠** — 禁止两人共用同一语气词
4. **信息量不对称** — 腹黑知道的比说的多，傻白甜说的比知道的多
5. **遮名测试** — 遮住名字，仅看对话能区分谁在说话

## 角色规则 vs 题材规则冲突处理

优先级：核心红线 > 人物底线 > 题材硬规则 > 剧情需要

| 冲突 | 处理 |
|------|------|
| "绝不撒谎" + 卧底需求 | 改标签或加"被逼到绝境"铺垫 |
| 社恐 + 大量社交场景 | 社恐本身是戏剧性来源，写焦虑而非跳过 |
| 角色底线与剧情矛盾 | 剧情让步，或设计高代价的弧线转折 |
