# 小说总谱

> AI 维护的长篇项目状态管理中心。与 `outputs/小说书写/` 互补——outputs 放成品，这里放元数据。

---

## 与 outputs 的关系

| 目录                | 职责                                                                |
| ------------------- | ------------------------------------------------------------------- |
| `outputs/小说书写/` | 正式创作成品（章节、大纲、角色卡、世界观）                          |
| `小说总谱/`         | AI 维护的项目状态（人物档案、势力档案、时间线、伏笔追踪、章节规划） |

写正文时优先写入 `outputs/`；需要维护长期设定时同步更新 `小说总谱/`。

---

## 项目模板

新建长篇项目时，在 `小说总谱/` 下创建：

```text
小说总谱/{小说名}/
├── README.md           ← 项目总览（一句话梗概、当前进度、下一章要做什么）
├── 人物档案.md          ← 所有角色的当前状态（存活/死亡/失踪/黑化/关系变化）
├── 势力档案.md          ← 各势力的当前状态（实力变化、成员变动、敌对关系）
├── 时间线.md            ← 按时间顺序的事件记录（每章关键事件 + 时间戳）
├── 伏笔追踪.md          ← 已埋伏笔 + 状态（待回收/已回收/废弃）
├── 章节规划.md          ← 未来3-5章的简要规划
└── 世界观状态.md         ← 世界观规则的当前状态（已揭示/未揭示/规则变化）
```

---

## 更新时机

| 事件                   | 更新内容                                 |
| ---------------------- | ---------------------------------------- |
| 每章写完               | 更新人物档案、时间线、伏笔追踪、章节规划 |
| 角色死亡/黑化/关系变化 | 更新人物档案                             |
| 势力变动               | 更新势力档案                             |
| 世界观规则揭示         | 更新世界观状态                           |
| 新伏笔埋下             | 更新伏笔追踪                             |
| 旧伏笔回收             | 标记伏笔状态为"已回收"                   |

---

## 记忆卡格式

每个项目根目录下的 `README.md` 作为"记忆卡"，格式：

```markdown
# {小说名} · 记忆卡

## 一句话梗概

（一句话说清楚这个故事讲什么）

## 当前进度

- 已完成：第X章
- 当前章节：第X章「章节名」
- 总字数：约X万字

## 人物当前状态

| 角色 | 状态 | 位置 | 当前目标 |
| ---- | ---- | ---- | -------- |
| 主角 | 存活 | XX   | XX       |

## 下一章要做什么

（3-5句话，写下一章时直接读这里）

## 待解决问题

- [ ] 伏笔A（第X章埋下）
- [ ] 角色B的行踪

## 上次更新时间

YYYY-MM-DD
```
