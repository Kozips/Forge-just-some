# 地图编辑器使用指南

> 通用地图编辑器，可嵌入任何角色扮演项目。支持多层地图、迷雾模式、动态添加地点。

---

## 文件位置

- 通用模板：`工具/地图编辑器/地图编辑器.html`（基础引擎，需要注入数据）
- 每个角色扮演项目生成的地图：`outputs/角色扮演/[项目名]/地图/地图编辑器.html`（数据内嵌，直接打开可用）

---

## 在角色扮演中的使用

### 开局生成地图

角色扮演启动时，根据世界观设定自动生成地图HTML，写入项目目录：

```
outputs/角色扮演/[项目名]/
└── 地图/
    └── 地图编辑器.html      ← 数据内嵌，直接浏览器打开
```

**生成规则**：
1. 从世界观设定中提取已知地点
2. 按类型分类（安全区/危险区/资源点/未知区等）
3. 角色初始位置标记为 `d:1`（已发现）
4. 角色未知的地点标记为 `d:0`（迷雾状态，显示为???）
5. 角色完全未知的地点不放入地图（等剧情触发后添加）

### 数据格式（内嵌在HTML的DATA数组中）

```javascript
var DATA = [
  {name:"🗺️ 地面", bg:"#16213e", cats:[
    {n:"安全区", c:"safe", bg:"#1b3a1b", bd:"#4caf50", ic:"🛡️"},
    {n:"危险区", c:"danger", bg:"#3a1b1b", bd:"#f44336", ic:"☠️"}
  ],items:[
    {id:"gas", c:"safe", t:310, l:380, w:140, h:65,
     html:"🛢️ 加油站<br><font size=1>📍当前位置</font>", d:1},
    {id:"hosp", c:"resource", t:100, l:350, w:150, h:70,
     html:"🏥 医院<br><font size=1>传闻有药品</font>", d:0}
  ]}
];
```

**字段说明**：

| 字段 | 含义 | 示例 |
|------|------|------|
| `name` | 标签页名称 | `"🗺️ 地面"` |
| `bg` | 地图背景色 | `"#16213e"` |
| `cats` | 建筑类别定义 | `[{n,c,bd,bg,ic}]` |
| `items` | 建筑数据 | `[{id,c,t,l,w,h,html,d}]` |
| `id` | 唯一标识 | `"gas"` |
| `c` | 类别class名 | `"safe"` |
| `t/l/w/h` | 位置和大小（数字，px） | `310, 380, 140, 65` |
| `html` | 显示内容（支持HTML） | `"🛢️ 加油站<br>..."` |
| `d` | 发现状态：1=已发现，0=迷雾 | `1` |

## 角色扮演中的动态更新

### LLM更新方式

角色扮演对话中，LLM直接修改HTML文件中的DATA数组，用户刷新页面看到更新。

⚠️ **关键**：每次修改DATA后，必须同时把localStorage key版本化（`fth-map` → `fth-map-v2` → ...），否则浏览器缓存的旧数据会覆盖新数据。详见踩坑记录·错误7。

### JS API

| API | 作用 | 示例 |
|-----|------|------|
| `addLoc(tabIndex, {id,c,t,l,w,h,html,d})` | 添加新地点 | `addLoc(0, {id:'x1',c:'safe',t:200,l:300,w:100,h:50,html:'🔥 营地',d:0})` |
| `discLoc(id)` | 发现地点（解除迷雾） | `discLoc('hosp')` |

### 事件对照

| 事件 | 操作 | JS调用 |
|------|------|--------|
| 角色到达新地点 | 解除迷雾 | `discLoc('hosp')` |
| 角色得知新地点 | 添加迷雾标记 | `addLoc(0, {..., d:0})` |
| 地点被摧毁 | 删除（手动在编辑器里删） | 用户操作 |
| 新区域解锁 | 新增标签页 | 修改DATA数组增加tab |

### 用户手动操作

- **拖拽**：移动建筑位置
- **右下角拖拽**：改变建筑大小
- **双击**：编辑建筑名称
- **Delete**：删除选中的建筑
- **📤 导出**：下载JSON文件（LLM可读取）
- **📥 导入**：加载JSON文件

## 踩坑记录（开发时的错误总结）

### ❌ 错误1：JSON格式漏引号

**症状**：页面空白，控制台报 `Unexpected token` 错误

**原因**：categories数组里漏了一个引号：
```json
// ❌ 错误："cls: "safe" 少了左边的引号
{ "name": "安全区", "cls: "safe", "bg": "#1b3a1b" }

// ✅ 正确
{ "name": "安全区", "cls": "safe", "bg": "#1b3a1b" }
```

**教训**：JSON里每个key都必须有完整的双引号，复制粘贴时最容易漏。

---

### ❌ 错误2：localStorage旧数据冲突

**症状**：地图显示了旧项目的数据，或不显示任何数据

**原因**：之前校园地图用的localStorage key跟新项目冲突，浏览器缓存了旧数据

**教训**：
- 每个项目的 `projectKey` 必须唯一（如 `"废土猎奇-map"`）
- 提供🔄重置按钮让用户清除旧数据
- 新项目首次打开时，如果显示异常，先点重置

---

### ❌ 错误3：通用引擎过于复杂导致不显示

**症状**：第一版通用地图编辑器（`工具/地图编辑器/地图编辑器.html`）打开后什么都没有

**原因**：
- 通用引擎用了复杂的CONFIG对象嵌套结构（tabs → categories → buildings → 多层属性名）
- CSS通过JS动态注入，如果CONFIG解析失败样式也跟着挂
- 事件绑定依赖DOM元素存在，元素没渲染成功事件也绑不上
- 一个JSON语法错误会导致整个引擎崩溃

**教训**：
- **数据结构越简单越好**——用 `c:"safe"` 不用 `cls:"safe"`，用 `t:310` 不用 `top:"310px"`
- **CSS直接写在style标签里**——不要通过JS动态注入
- **数据内嵌在HTML里**——不要依赖外部JSON文件加载（浏览器的CORS会拦本地file://请求）
- **短变量名减少出错概率**——`t,l,w,h,d` 比 `top,left,width,height,discovered` 不容易拼错

---

### ❌ 错误4：建筑位置用字符串"310px"而不是数字310

**症状**：建筑位置计算出错，拖拽后位置不对

**原因**：第一版用字符串 `"top": "310px"`，JS里做加减法时字符串拼接而不是数学运算

**教训**：
- 数据里存数字：`t:310`
- 渲染时加px：`el.style.top = it.t + 'px'`
- 拖拽计算用 `parseInt(el.style.top)`

---

### ❌ 错误5：迷雾状态判断用布尔值在HTML dataset里不可靠

**症状**：迷雾模式切换不生效

**原因**：`el.dataset.d = false` 在HTML里变成字符串 `"false"`，`=== false` 判断永远不成立

**教训**：
- 用数字 1/0 而不是 true/false
- 判断时用 `el.dataset.d === '1'` 而不是 `el.dataset.d === true`
- 或者用 `el.dataset.d !== '0'` 来判断"已发现"

---

### ❌ 错误6：innerHTML里嵌套的HTML标签被resize-handle干扰

**症状**：双击编辑后，建筑名称变成了乱码或丢失格式

**原因**：innerHTML里包含 `<div class="rh">` 这个拖拽手柄，编辑时把它也算进去了

**教训**：
- 编辑时先移除resize-handle：`el.innerHTML.replace(/<div class="rh"><\/div>/, '')`
- 保存时只存建筑内容，不存resize-handle
- resize-handle通过JS动态添加，不写在数据里

---

### ❌ 错误7：LLM更新内嵌数据后刷新页面不生效

**症状**：LLM通过Edit工具修改了HTML文件中的DATA数组（新增地点、修改状态等），用户刷新浏览器后地图没有变化，仍显示旧数据

**原因**：`load()` 函数优先从localStorage读取数据并覆盖内嵌的DATA：
```javascript
// load() 在页面初始化时调用，用localStorage的旧数据替换文件里的新数据
var raw = localStorage.getItem('fth-map');
if (!raw) return;
var saved = JSON.parse(raw);
saved.forEach(function(t, i) { if (DATA[i]) DATA[i].items = t.items; });
```
用户之前打开过地图 → localStorage有缓存 → 文件改了也没用，localStorage赢。

**教训**：
- **方案A（推荐）：版本化localStorage key** — 每次LLM更新内嵌数据时，把key从 `fth-map` 改成 `fth-map-v2`、`fth-map-v3`...旧缓存自动失效，文件数据生效
- **方案B：删掉load()函数** — 如果不需要跨会话保存用户手动调整的位置，直接去掉localStorage读取逻辑
- **方案C：加时间戳优先级** — 在DATA里加一个 `_ts` 时间戳字段，load()时比较，文件比localStorage新就用文件的
- **记住**：localStorage是浏览器行为，LLM改文件不会清localStorage，两个数据源会打架
