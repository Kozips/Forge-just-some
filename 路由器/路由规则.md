# 路由规则

> 用户输入 → 解析意图 → 按优先级加载对应skill → 输出

---

## 路由流程

```
用户输入
  ↓
【1】意图解析：用户要什么？
  → 写小说？→ 进入写作路由
  → 扮演角色？→ 进入角色扮演路由
  → 查设定？→ 进入设定查询路由
  → 混合？→ 按权重分配
  ↓
【2】题材识别：写的什么类型？
  → 从写作/类型/目录匹配
  → 支持多题材叠加（如：恐怖+悬疑）
  ↓
【3】世界观匹配：什么背景？
  → 现实世界？→ 加载国家拟实
  → 架空世界？→ 加载世界观模板
  → 混合？→ 先加载国家拟实，再叠加架空设定
  ↓
【4】文化设定加载：需要什么体系？
  → 根据题材+背景自动匹配（如都市类→官场+商业）
  ↓
【5】人物实例化：角色是谁？
  → 人物类+性格类+职业类组合
  → 经过兼容矩阵检查
  ↓
【6】文风+对话习惯：怎么写？
  → 根据人物+题材匹配文风
  → 加载对应对话习惯模块
  ↓
【7】输出：生成内容
```

---

## 路由匹配表

| 用户输入关键词 | 路由目标 | 加载内容 |
|--------------|---------|---------|
| "写一个都市小说" | 写作路由 | 都市类skill + 官场/商业文化设定 + 龙国拟实 |
| "扮演一个社恐女大学生" | 角色扮演路由 | 人物类/学生型 + 性格类/社恐 + 对话习惯 |
| "帮我写一个恐怖故事" | 写作路由 | 恐怖类skill + 对话习惯 |
| "韩国财阀背景的小说" | 写作路由 | 都市类 + 财阀体系 + 泡菜国拟实 |
| "古代武侠" | 写作路由 | 玄幻类 + 冷兵器 + 架空世界观 |
| "写一个霸道总裁" | 写作路由 | 情感类 + 商业体系 + 人物类/恋爱型 |

---

```
3. 文化设定：设定参考/文化设定/商业体系.md
4. 国家拟实：设定参考/各国家拟实/龙国/社会生活.md
7. 对话习惯：人物/references/research/对话习惯/类型对话/08_情感对话引擎.md

【冲突检查】
- 无冲突

【开始创作】
```
