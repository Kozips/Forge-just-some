# 输出规则

> 从 `forge/SKILL.md` 迁出。根入口只保留一句输出去向；详细规则放在这里。

---

## 默认输出位置

所有创作作品统一放在 `outputs/` 目录下（此目录不在开源仓库中，由用户自行创建）。

| 作品类型 | 路径 | 说明 |
| -------- | ---- | ---- |
| 单篇短文 | `outputs/小文章/` | 散文、短篇故事、评论、单章片段 |
| 多章小说 | `outputs/小说书写/[小说名]/` | 所有多章小说统一使用此路径 |

---

## 多章小说结构

每部新小说必须创建以下完整子目录：

```text
outputs/小说书写/{小说名}/
├── 大纲/          ← 全书大纲、章节规划、时间线
├── 章节/          ← 正文章节（第01章.md, 第02章.md ...）
├── 人物设定/      ← 角色卡、人物档案、关系图
├── 世界观/        ← 世界观设定、规则体系、地图
└── 番外/          ← 番外篇、支线、彩蛋
```

**⚠️ 新建小说时必须一次性创建全部5个子目录，不要只建章节目录。**

---

## 与小说总谱的区别

- `outputs/`：正式创作成品。
- `小说总谱/`：AI 维护的项目状态，如人物档案、势力档案、时间线、伏笔追踪、章节规划。

写正文时优先写入 `outputs/`；需要维护长期设定时再同步更新 `小说总谱/`。

---

## 章节输出节奏控制（硬性规则）

> 避免单次输出过长导致 API 超时。每章写完即停，交由用户决定是否继续。

**规则：**

1. **一章一停**：每写完一章（2000-3000字），立即结束当前轮次，不要自动续写下一章。
2. **写后询问**：章节写完存入文件后，输出简短的本章摘要（3-5句话），然后问用户："要继续写下一章吗？"
3. **确认后续**：用户说继续，就在当前对话里接着写下一章。用户说停，就结束。
4. **上下文传递**：每章结尾附带一段"续写上下文"（人物当前状态、情节进度、待解决伏笔），方便后续衔接。
5. **超时兜底**：
   - 单章正文控制在 **2000-3000字** 以内。不要贪长。
   - 超过3000字的场景，拆成独立章节。
   - 写之前不要做大量内部规划，直接动笔用Write工具输出。
   - 一次Write调用写完一整章，不要分多次拼接。

**格式示例（写完一章后输出）：**

```
✅ 第X章「章节名」已写入：outputs/小说书写/{小说名}/第X章_章节名.md

📝 本章摘要：xxx

🔖 续写上下文：
- 人物状态：xxx
- 情节进度：xxx
- 待解决伏笔：xxx

要继续写下一章吗？
```
