# 滴滴

> 中国最大的网约车平台，覆盖快车、专车、优享、拼车、代驾、顺风车等多种出行方式。是都市出行类小说和深夜故事的天然舞台。

---

## 一、平台定位（写作用途）

- **国民出行平台**：一个人人都用过的打车工具，连接司机与乘客的移动空间
- **用户画像**：20-50岁城市居民，商务人士、加班族、夜归人、酒后出行者、机场高铁旅客
- **气质标签**：都市感、深夜、密闭空间、陌生人对话、流动的城市切面、出行焦虑
- **写作适用**：深夜打车故事、司机与乘客的偶遇/对峙、职场加班夜归、酒后失控、安全事件、城市空间的流动性叙事、陌生人之间的短暂交集

---

## 二、核心功能

### 2.1 打车产品线

| 产品 | 定位 | 价格 | 写作场景 |
|------|------|------|---------|
| **特惠快车** | 最便宜的快车，一口价 | 最低 | 省钱/学生/预算紧张 |
| **快车** | 最常用的打车方式 | 中等 | 日常通勤/日常出行 |
| **优享** | 比快车稍好，车况和司机评分更高 | 中高 | 商务/约会/不想太寒碜 |
| **专车** | 高端出行，商务级别 | 高 | 商务接待/重要场合/有钱人 |
| **拼车** | 多人共享一车，最便宜 | 最低 | 省钱/不赶时间/社交偶遇 |
| **顺风车** | 私家车主顺路带人 | 很低 | 跨城/长途/省钱出行 |
| **代驾** | 替喝了酒的人开车 | 按距离收费 | 酒局后/商务应酬 |

### 2.2 打车流程（写作细节参考）

**乘客端**：
输入起终点→选择车型→叫车→等待接驾→司机接驾→上车→行驶中→到达目的地→支付→评价

**司机端**：
听单/抢单→导航前往乘客位置→接驾→确认上车→导航前往目的地→到达→乘客支付→收到评价

### 2.3 司机生态（核心写作素材）

**司机类型**：
- **全职司机**：每天跑10-14小时，月入6000-12000（扣除平台抽成和油费/电费后）
- **兼职司机**：下班后或周末跑几单，补贴家用
- **夜班司机**：专跑夜间（22:00-6:00），单价高但乘客状态复杂
- **代驾司机**：酒后代驾，骑折叠电动车去接单，开客人的车送回去

**司机评分体系**：
- 乘客和司机双向评分（1-5星）
- 司机评分低于一定阈值会被限制接单甚至封号
- 高分司机优先接到优质订单（长途、高单价）
- 差评/投诉对司机影响极大——一次投诉可能扣几百块

**司机日常**：
- 平台抽成约20-30%（实际到手比乘客付的少很多）
- 早晚高峰有冲单奖励（完成X单额外奖励）
- 需要自备车辆或租车（租车月费3000-5000）
- 导航、路况、乘客素质是三大变量

### 2.4 安全功能（2018年安全事件后强化）

| 功能 | 说明 |
|------|------|
| **行程分享** | 一键分享实时行程给微信好友 |
| **紧急求助** | 一键报警，自动发送位置给警方 |
| **录音保护** | 行程全程录音（默认开启） |
| **虚拟号码** | 司机和乘客互相看不到真实手机号 |
| **人脸识别** | 司机接单前需刷脸验证身份 |
| **夜间安全** | 深夜打车有额外安全提示 |
| **亲友守护** | 可设置紧急联系人，行程异常自动通知 |

### 2.5 特色功能

- **一口价**：特惠快车/拼车的固定价格，堵车不加钱
- **顺风车**：车主发布行程，乘客匹配搭乘，社交属性最强
- **接驾**：司机到达乘客位置后等待的过程（有免费等待时长）
- **目的地**：司机可以设置目的地，只接顺路单
- **预约单**：提前预约用车时间（机场/高铁出行常用）

## 四、写作场景模板（可直接嵌入剧情）

### 场景1：深夜打车
> 凌晨一点，她站在KTV门口叫了一辆特惠快车。上车以后报了手机尾号，然后就靠在后座上闭眼。车里很安静，只有导航的机械女声。开了五分钟，司机从后视镜里看了她一眼："姑娘，你哭了吗？"她这才发现自己脸上有泪痕，用手背擦了一下，说了句"没事"。司机没再问，把车内暖风调高了一点，默默开完了全程。

### 场景2：拼车偶遇
> 他上车的时候发现后座已经坐了一个人——一个抱着笔记本电脑敲键盘的女生。他犹豫了一下坐到了副驾驶。开到一半，司机又接了一单，后座变成三个人。女生的电脑快没电了，他正好带了充电宝，递过去说"Type-C的，能用吗？"她抬头看了他一眼："谢谢。"下车的时候两人互加了微信，谁也没说这是不是"搭讪"。

### 场景3：司机的日常
> 老刘早上七点出车，跑了一上午才赚了一百出头。中午在路边买了个煎饼果子，坐在车里吃完继续听单。下午三点接到一个长途——从城东到机场，这一单能赚八十多。乘客上车以后一直在打电话谈生意，声音很大。老刘安静地开车，把导航声音调小了一点。到了机场，乘客匆匆下车，打了个五星，没有小费。老刘看了一眼时间，还来得及在机场听一单返程的。

### 场景4：代驾深夜
> 他骑着折叠电动车到饭店门口，客户已经醉得站不稳了。"哥，车钥匙给我，我送您回去。"他把折叠电动车塞进后备箱，调好座椅和后视镜，发动了客户的奔驰。后座的客户含糊不清地报了一个地址，然后就睡着了。他看着导航上十五公里的路程，把车速控制在限速以内，安安静静开完了全程。到了以后叫醒客户，对方掏出手机扫码付了款，嘟囔了一句"谢谢啊"，踉跄着进楼了。他把电动车从后备箱拿出来，骑进夜色里，接下一单。

### 场景5：安全事件
> 她上车以后发现司机走的不是导航路线，心跳一下加速了。她悄悄点开滴滴的行程分享发给闺蜜，又按了紧急求助。手指悬在报警按钮上方，正要按下，司机说话了："前面修路，我绕一下，多走两公里，但更快。"她看了看窗外——确实有围挡。松了一口气，取消了紧急求助，但还是把行程分享的链接留在了微信里，一直到下车才关掉。

### 场景6：凌晨的顺风车
> 她在顺风车平台上发了一单——明天早上六点从学校到高铁站。很快有车主接单了，头像是个中年男人，车是本田雅阁，评分4.9。她点进去看了一下评价——"准时""车干净""人很nice"。犹豫了一下，还是上车了。车主是个出差的销售，正好顺路去高铁站。一路上聊了很多——工作、城市、未来。到站以后她说了声谢谢，头也没回地冲进了候车厅。后来她再也没有打到过同一辆车。

## 五、写作避坑

1. **不要把司机写成"工具人"**——司机是有生活、有故事的人。写司机时要给背景、给性格、给情绪，不要只是一个"会开车的NPC"
2. **拼车不是"艳遇平台"**——拼车写作不要每一次都写成偶遇暧昧，真实的拼车更多是安静、各玩各的手机、偶尔尴尬的沉默
3. **2018年安全事件是敏感话题**——写滴滴安全相关情节时要谨慎处理，不要消费真实事件，可以写"安全焦虑"但不要写成猎奇
4. **平台抽成是司机的真实痛点**——"乘客付了50，司机到手35"这种真实的抽成比例可以写，能让读者理解司机的辛苦，但不要过度渲染仇恨
5. **深夜场景要用对氛围**——深夜打车天然带悬疑/暧昧/孤独气质，但不要每次都往"危险"方向写。很多深夜打车的故事其实是安静的、温暖的、甚至无聊的——这种真实感比戏剧性更有力量
6. **一口价和快车的价格差异要写对**——特惠快车/拼车是一口价（固定价格），快车/优享/专车是计价器（根据路程和时间计算）。写"堵车加钱"的场景只能发生在非一口价产品上
